🛡️ Model Robustness Evaluation ModuleThis module evaluates model performance and stability under real-world input variations, identifying failure patterns and performance degradation without relying on external LLM dependencies for perturbations.📖 Table of ContentsObjectiveArchitecture OverviewProject StructurePerturbation TypesSlice DefinitionsHow to RunExecution FlowOutput ArtifactsDesign Principles🎯 ObjectiveThe module tests model robustness against:Input Variations: Typos, noise, truncation, and restructuring.Stability: Paraphrasing and case normalization.Key Goals:Identify performance degradation (Delta vs. Baseline).Surface weak slices within the dataset.Log failure patterns and specific regression cases.🧩 Architecture OverviewThe pipeline follows a linear data flow to ensure efficiency:Input Dataset → Perturbations (5 types) → Model Inference (Single Load) → Evaluation (Metrics + Taxonomy) → Slice Analysis → Artifact Generation📁 Project StructurePlaintextsrc/robustness/

⚙️ Perturbation TypesTypeDescriptionImpactTypo InjectionChar swaps, deletions, keyboard errors (~5% rate).Simulates human typing errors.Noise InjectionAdds irrelevant prefix text.Tests ability to ignore distractions.TruncationRemoves the last 30% of input.Simulates cut-off or incomplete inputs.Case NormalizationConverts all input to lowercase.Tests reliance on capitalization signals.Sentence ShuffleRandomizes sentence order.Tests sensitivity to structural flow.🧪 Slice DefinitionsEach sample is automatically categorized into slices for granular analysis:Archetype: single_step, multi_step, refusal.Dataset Source: Derived from metadata.source (defaults to unknown).Input Length: Short, Medium, Long.Complexity: * Simple (1 step)Moderate (2–3 steps)Complex (4+ steps)🚀 How to RunFull EvaluationBashpython scripts/run_robustness.py \
    --config configs/eval/json_eval.yaml \
    --checkpoint outputs/checkpoints/best_model
Run Specific PerturbationsBashpython scripts/run_robustness.py \
    --config configs/eval/json_eval.yaml \
    --checkpoint outputs/checkpoints/best_model \
    --perturbations typo noise truncation
🔄 Execution FlowInitialize: Load model into memory once (load_model_for_inference).Baseline: Run inference on the clean dataset.Perturb: Apply the 5 perturbation types to the dataset.Evaluate: * Run run_all_metrics().Classify errors via error_taxonomy.Analyze: Compute slice-level metrics and Delta vs. Baseline.Export: Generate reports and regression logs.📊 Output ArtifactsReports are generated in outputs/reports/robustness/<checkpoint_name>/.clean_metrics.json: The baseline performance.robustness_summary.json: A delta comparison table highlighting the worst-performing perturbations.slice_report.json: Metrics broken down by Archetype, Length, and Complexity.failure_log.json: Detailed log of VALID → FAIL regressions.robustness_report.md: A human-readable summary of key findings and insights.📉 Key MetricsAccuracy Drop: Absolute change in success rate.Structural/Functional Degradation: Changes in output formatting or logic.Consistency: Stability of answers under paraphrasing.Failure Transition Rate: Frequency of correct inputs becoming incorrect after perturbation.🧠 Design PrinciplesEfficiency: Single model load for all perturbation tests.Extensibility: Modular perturbation classes for easy additions.Reproducibility: Deterministic slicing and shuffling.Actionability: Taxonomy-driven logs that tell you why a model failed, not just that it failed.🛠️ Future Extensions[ ] LLM-based paraphrase perturbations.[ ] Adversarial input generation.[ ] Real-time user log evaluation.[ ] Integration with Phase 4 Robustness Dashboards.
